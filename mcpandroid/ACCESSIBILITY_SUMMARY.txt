ACCESSIBILITY FEATURES IMPLEMENTED:
===================================

✓ Screen Reader Support
  - All interactive elements have accessibilityLabel
  - Proper accessibilityRole assignments (button, header, alert, text)
  - accessibilityHint for complex actions
  - accessibilityState for disabled/busy states

✓ Touch Target Sizes (44x44pt minimum)
  - Copy buttons: hitSlop={{ top: 12, bottom: 12, left: 12, right: 12 }}
  - Refresh buttons: hitSlop={{ top: 10, bottom: 10, left: 10, right: 10 }}
  - All buttons meet iOS HIG (44pt) and Android Material (48dp) requirements
  - Visual appearance UNCHANGED (touch area expanded invisibly)

✓ Live Regions for Dynamic Content
  - Server status changes: accessibilityLiveRegion="polite"
  - Error messages: accessibilityLiveRegion="assertive"
  - Loading states: accessibilityState={{ busy: true }}
  - Log updates announced automatically

✓ Semantic HTML/ARIA Equivalents
  - Headers marked with accessibilityRole="header"
  - Alerts use accessibilityRole="alert"
  - Progress indicators use accessibilityRole="progressbar"
  - Buttons properly identified

✓ Content Grouping
  - Related info grouped to reduce verbosity
  - Decorative elements hidden from accessibility tree
  - Redundant text marked with accessibilityElementsHidden
